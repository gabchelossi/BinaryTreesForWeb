BinaryTreesForWeb — Command and Animation Help

This page documents the terminal commands available in the Binary Search Tree and AVL Tree visualizer. The terminal is a custom command interface for interacting with the tree; it is not an operating-system shell.

Most commands have a direct graphical equivalent in the control panel. You can either type commands in the terminal or use the buttons, radio buttons, checkboxes, and slider in the interface.

Project Overview

BinaryTreesForWeb is an interactive visualizer for Binary Search Trees and AVL Trees. It allows users to insert, remove, search, traverse, rebalance, and inspect tree structures directly in the browser.

The project is designed for learning. It shows both the visual tree and the internal array representation, making it easier to understand how tree shape affects storage, height, and operation cost.

Command and GUI Equivalence

The terminal and the graphical control panel are two interfaces for the same underlying tree operations. The terminal is useful for quickly executing precise sequences. The GUI is useful for exploring the same operations visually.

Animation Visual Guide

The animation system is not only decorative. It is used to expose the algorithm's state while the operation is running. Colors indicate whether a node is being considered, has been accessed, has been visited, or has been identified as part of a rebalancing process.

Traversal Animation Colors

During animated traversals, the visualizer distinguishes between nodes that have not yet been visited, nodes that are currently being accessed, and nodes that have finally been visited by the traversal.

Red: the node exists in the traversal area but has not been finally visited yet. It is still pending according to the selected traversal order.
Yellow / orange: the node or connection is currently being accessed. This represents the algorithm's active position while it explores the tree.
Green: the node has been finally visited. At this point, the node has been appended to the traversal result.

Search Animation

Animated search follows the same decision process as a standard Binary Search Tree lookup. Starting at the root, the algorithm compares the target key with the current node:

  1. If the target is equal to the current node, the search succeeds.
  2. If the target is smaller, the algorithm moves to the left child.
  3. If the target is larger, the algorithm moves to the right child.
  4. If the next position is empty, the search fails because the key is not present.

During animation, the currently explored node is highlighted as the algorithm moves through the tree. When the key is found, the final matching node remains highlighted to show the result.

AVL Rebalancing Animation

In AVL mode, insertion and removal can change subtree heights. After the structural BST operation is complete, the visualizer checks the path back toward the root and updates AVL metadata.

The displayed red values around a node represent AVL-related measurements: subtree heights/weights and balance information. The balance is used to determine whether a node has become unbalanced.

  1. The traverser starts near the node affected by the insertion or deletion and moves upward through the ancestors.
  2. For each ancestor, the visualizer recomputes the left and right subtree information.
  3. If the balance remains within the AVL limit, the traverser continues upward.
  4. If a node becomes unbalanced, that node is treated as z, the first unbalanced ancestor.
  5. The taller child of z is selected as y.
  6. The child of y that determines the rotation case is selected as x.
  7. The visualizer then performs the appropriate AVL rotation: LL, RR, LR, or RL.

In short: animated AVL mode shows not only that the tree rebalances, but also how the algorithm decides where the imbalance is, which nodes define the rotation, and how the final balanced subtree is produced.

Array Representation

The tree is represented internally using an array. The root is stored at rank 0. For a node at rank i:

Relationship Formula
Left child 2*i + 1
Right child 2*i + 2
Parent floor((i - 1) / 2)

This representation intentionally highlights a weakness of unbalanced BSTs. A right-heavy tree such as insert 1 2 3 4 5 may use ranks 0, 2, 6, 14, and 30, leaving many empty positions.

Use show array true to display the full internal array, including empty slots.

Insertion Commands

insert [value]

Inserts one integer key into the tree.

guest@gchelossi-dev.com: insert 42

insert [v1] [v2] [v3] ...

Inserts multiple keys in the order provided.

guest@gchelossi-dev.com: insert 50 30 70 20 40 60 80

insert full

Resets the current tree and inserts a predefined full test tree. This is useful for quickly generating a large example. 

guest@gchelossi-dev.com: insert full

insert random

Inserts 20 unique random keys between 0 and 99. The median value is inserted first so the initial root is centered. 

guest@gchelossi-dev.com: insert random

insert random [max]

Inserts 20 unique random keys between 0 and max.

guest@gchelossi-dev.com: insert random 200

insert random [min] [max]

Inserts 20 unique random keys between min and max.

guest@gchelossi-dev.com: insert random 10 80

insert random [min] [max] [count]

Inserts count unique random keys between min and max. The count cannot be larger than the size of the range. 

guest@gchelossi-dev.com: insert random 10 80 30

Removal Command

remove [value]

Removes a key from the tree if it exists. The deletion logic supports standard BST removal cases: leaf deletion, deletion of a node with one child, deletion of a node with two children, and root deletion.

When AVL mode is active, removals may also trigger AVL rebalancing. 

guest@gchelossi-dev.com: remove 42

Search Command

search [value]

Searches for a key and returns its rank in the array representation. If animations are enabled, the search path is highlighted visually. 

guest@gchelossi-dev.com: search 30

Traversal Commands

traverse in-order or traverse inorder

Visits nodes in sorted order: left subtree, root, right subtree.

guest@gchelossi-dev.com: traverse in-order

traverse pre-order or traverse preorder

Visits nodes in this order: root, left subtree, right subtree.

guest@gchelossi-dev.com: traverse pre-order

traverse post-order or traverse postorder

Visits nodes in this order: left subtree, right subtree, root.

guest@gchelossi-dev.com: traverse post-order

AVL Commands

set avl on

Activates AVL mode. Future insertions and removals automatically rebalance the tree. If the tree already contains nodes, the visualizer rebuilds the current key set using AVL insertion rules. 

guest@gchelossi-dev.com: set avl on

set avl off

Deactivates AVL mode. Future insertions behave like a regular, non-self-balancing Binary Search Tree. 

guest@gchelossi-dev.com: set avl off

Supported AVL Rotation Cases

Case Meaning
LL Left-left imbalance
RR Right-right imbalance
LR Left-right imbalance
RL Right-left imbalance

Animation Commands

set animation on

Enables automatic animations.

guest@gchelossi-dev.com: set animation on

set animation off

Disables animations. Operations execute immediately.

guest@gchelossi-dev.com: set animation off

set animation manual

Enables step-by-step animation mode. The next-step action is controlled through the visual animation button in the interface. 

guest@gchelossi-dev.com: set animation manual

set animation speed [1-5]

Sets the animation speed. The speed must be an integer between 1 and 5. Higher values make the animation faster. 

guest@gchelossi-dev.com: set animation speed 3

pause

Pauses the animation system.

guest@gchelossi-dev.com: pause

play

Resumes the animation system.

guest@gchelossi-dev.com: play

Display Commands

show array

Displays the internal array representation, showing only initialized real nodes.

guest@gchelossi-dev.com: show array

show array true

Displays the full array representation, including empty slots. This is useful for seeing how sparse the array becomes in an unbalanced tree. 

guest@gchelossi-dev.com: show array true

show empty on

Displays visual empty nodes at uninitialized child positions. These placeholders show where future insertions would go. 

guest@gchelossi-dev.com: show empty on

show empty off

Hides visual empty nodes.

guest@gchelossi-dev.com: show empty off

show help

Displays help for the show command.

guest@gchelossi-dev.com: show help

Utility Commands

equivalent

Returns an equivalent insert ... command containing the keys currently stored in the tree. This is useful for reproducing the current tree later. 

guest@gchelossi-dev.com: equivalent

reset

Clears the tree and removes all nodes and connections. This is equivalent to using the Reset button in the control panel. 

guest@gchelossi-dev.com: reset

clear

Clears the terminal output.

guest@gchelossi-dev.com: clear

credits

Displays developer credits and contact information.

guest@gchelossi-dev.com: credits

help

Displays help information inside the terminal and links to this full help page.

guest@gchelossi-dev.com: help

Example Workflows

Build a regular BST

guest@gchelossi-dev.com: set avl off
guest@gchelossi-dev.com: insert 50 30 70 20 40 60 80
guest@gchelossi-dev.com: show array true
guest@gchelossi-dev.com: traverse in-order

Build an AVL Tree

guest@gchelossi-dev.com: set avl on
guest@gchelossi-dev.com: insert 50 30 70 20 40 60 80
guest@gchelossi-dev.com: show array

Compare unbalanced BST behavior

guest@gchelossi-dev.com: reset
guest@gchelossi-dev.com: set avl off
guest@gchelossi-dev.com: insert 1 2 3 4 5 6 7 8 9
guest@gchelossi-dev.com: show array true

Compare AVL behavior

guest@gchelossi-dev.com: reset
guest@gchelossi-dev.com: set avl on
guest@gchelossi-dev.com: insert 1 2 3 4 5 6 7 8 9
guest@gchelossi-dev.com: show array true

Use random input

guest@gchelossi-dev.com: reset
guest@gchelossi-dev.com: set avl on
guest@gchelossi-dev.com: insert random 0 100 25
guest@gchelossi-dev.com: traverse in-order

Use manual animation

guest@gchelossi-dev.com: set animation manual
guest@gchelossi-dev.com: insert 30 20 10

In manual animation mode, use the animation control button in the interface to advance through the operation step by step.
